Virtue Foundation Dataset — Gap Analysis Guide

Columns and data types most useful for analyzing healthcare demand vs. availability in India. Based on exploration of the Databricks Virtue Foundation hackathon dataset (June 2026).

Essential for linking facility pincodes to NFHS districts.

Recommended Gap Analysis Approach

nfhs_5_district_health_indicators (demand) ↑ │ district + statename │ india_post_pincode_directory (bridge) ↑ │ address_zipOrPostcode = pincode │ facilities (supply) Per district: SUPPLY → facility count by type, beds/doctors, specialty coverage, public vs private mix DEMAND → disease burden %, unmet FP need %, low institutional birth %, care-seeking gaps GAP → demand score − normalized supply score

Example gap metrics (per district)

  1. Maternal care gap — low institutional_birth_5y_pct + low count of facilities with obstetrics/gynecology specialties
  2. Pediatric gap — high stunting/wasting rates + low pediatric-specialty facilities per child population
  3. Chronic disease gap — high diabetes/hypertension prevalence + low internal medicine/cardiology/endocrinology supply
  4. Geographic access gap — NFHS care-seeking rates vs facility density per 100k population (using lat/long for distance analysis)
  5. Public sector gap — high public-facility birth need vs low operatorTypeId = 'public' facility count

Important Caveats

  1. Coverage bias — Facilities skew heavily private (~88%) and urban (Maharashtra, Gujarat, UP top). This is not a complete census of India's healthcare system.
  2. Granularity mismatch — NFHS is district-level; facilities are point-level. Use the pincode bridge + spatial aggregation.
  3. String-typed capacity fields — numberDoctors and capacity need TRY_CAST; only ~25–36% have usable numeric values.
  4. NFHS string percentages — Many _pct columns need cleaning (*, parentheses, blanks).
  5. State name normalization — address_stateOrRegion sometimes contains district names (e.g. "Alappuzha" instead of "Kerala"); you'll need a mapping layer.
  6. Specialty arrays — specialties, procedure, equipment, and capability are JSON arrays stored as strings; explode/parse them for specialty-level gap analysis.

v1 Implementation — Hypertension vs. Cardiac Gap (June 2026)

Demand metric: w15_plus_with_high_bp_sys_gte_140_mmhg_and_or_dia_gte_90_mm_pct — the NFHS-5 share of women aged 15+ with high blood pressure (systolic ≥140 mmHg and/or diastolic ≥90 mmHg) in each district.

Gap: For each district we normalize hypertension prevalence (demand) and cardiac-specialty facility count (supply), then compute gap_score = demand_norm − supply_norm. Positive scores indicate districts where hypertension burden is high relative to cardiac care availability — not acute cardiac event rates.

v1 results: 706 NFHS districts analyzed; 96.2% of facilities map to a pincode; 406 districts have zero cardiac facilities; top gaps include Sikkim (South/North District), Sindhudurg (Maharashtra), and Pathanamthitta (Kerala). SQL lives in hackathon-app/config/queries/hypertension_gap_*.sql; dashboard on the Analytics tab.

See caveats section above — facility coverage is incomplete and urban-skewed.
