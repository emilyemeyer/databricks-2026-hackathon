Executive Summary

The Virtue Foundation dataset is usable for hackathon gap analysis but not analysis-ready in its raw form. Quality issues fall into four categories:

Category Severity Impact
Join fragility — state/district name mismatches, pincode fan-out, missing bridge rows Critical 40% of NFHS district-rows match zero facilities today; counts can be inflated by duplicate pincode joins
Schema inconsistency — numeric fields as strings, mixed NFHS types, JSON stored as text Critical Only 25–36% of capacity/doctor fields parse; many NFHS indicators need cleaning before math
Semantic errors — districts labeled as states, garbage in operatorTypeId, NFHS typos High State rollups and public/private splits are unreliable without normalization
Coverage bias — private/urban-heavy facility scrape vs nationally representative NFHS Medium Zero-facility districts may reflect real gaps or incomplete scraping — must be flagged, not silently treated as zero supply
Recommended approach: Do not mutate the source tables. Build a curated schema (virtue_foundation_curated) with cleaning pipelines, a reference geography dimension, and a data-quality scorecard table updated on each run.

Target Architecture

Introduce a curated layer alongside the raw hackathon tables. Raw tables remain read-only; all cleaning happens downstream.

Phased Improvement Plan

Phase 0 — Baseline & guardrails Week 1

Goal: Stop silent bad joins. Document what exists before changing anything.

  • Snapshot row counts, null rates, and distinct-value counts for all three raw tables.
  • Publish this plan and the gap analysis guide as team references.
  • Add a join_confidence column to any app query: matched, pincode_only, unmatched.
  • Mark raw tables read-only in documentation; route all analytics through curated views.

Exit criteria: Baseline metrics stored in dq_run_summary; no production query reads raw geography fields without normalization.

Phase 1 — Reference geography & join repair Week 1–2

Goal: Fix the highest-impact issue — broken district/state linkage.

  • Build ref_state_ut with canonical state names, aliases (Orissa→Odisha, Maharastra→Maharashtra, Tamilnadu→Tamil Nadu), and NFHS vs India Post variants.
  • Build ref_district_aliases for spelling variants (Ahmadabad→Ahmedabad, Allahabad→Prayagraj if applicable).
  • Build deduped bridge_pincode_district with documented tie-break: prefer HQ post office, then highest delivery status, then mode district by row count.
  • Produce dim_geography at NFHS grain (706 rows) with is_homonym flag for the 8 ambiguous district names.
  • Backfill facility district_id via pincode bridge; fallback to lat/long reverse lookup only when pincode fails.

Exit criteria: Strict district+state join match rate ≥ 75% of facilities; zero-facility NFHS districts drops from 283 to < 200 (excluding documented coverage gaps).

Phase 2 — Schema normalization & parsing Week 2–3

Goal: Make numeric and structured fields reliably computable.

  • Parse numberDoctors and capacity into doctors_count, bed_count (nullable INT) with parse_status flag.
  • Normalize operatorTypeId to enum: public, private, unknown; map government→public; quarantine garbage rows.
  • Parse JSON array columns into child tables: dim_facility_specialty, dim_facility_capability, etc.
  • Clean all NFHS string _pct columns: strip whitespace, parse parentheses, map * to NULL with is_suppressed = true.
  • Cast pincode directory lat/long to DOUBLE; validate India bounding box (lat 6–38, lon 68–98).

Exit criteria: ≥ 95% of NFHS indicator columns queryable as DOUBLE; ≥ 90% of facilities have valid enum operator type; JSON arrays explodable without parse errors on ≥ 99% of rows.

Phase 3 — Entity resolution & deduplication Week 3–4

Goal: One facility = one row in dim_facility.

  • Use cluster_id as primary dedup key; pick canonical row by most complete record (most non-null fields).
  • Secondary dedup: same name + pincode within 100m lat/long → flag as suspected duplicate.
  • Do not hard-delete; write quarantined rows to dim_facility_duplicates with merge rationale.

Exit criteria: Duplicate rate < 0.1%; dedup rules documented and auditable.

Phase 4 — Coverage enrichment & bias documentation Ongoing

Goal: Reduce false “zero supply” signals and set analyst expectations.

  • Flag districts with supply_coverage_tier: high, medium, low, none_observed based on facility density vs national median.
  • Add data_source_completeness_note to gap outputs: “supply side reflects scraped private facilities; not a government census.”
  • Optional: enrich with open government facility registries (NHRR, state health dept lists) where license permits.
  • Impute pincode from lat/long for the 462 facilities with orphan pincodes (reverse geocode).

Exit criteria: Every zero-supply district in gap output carries a coverage-confidence label; analysts cannot misread missing data as confirmed absence.

Phase 5 — Continuous quality monitoring Steady state

Goal: Prevent regression when source data refreshes.

  • Databricks Lakeflow job or DLT pipeline runs curation + DQ checks on schedule.
  • Alert when any DQ threshold breaches (see Validation section).
  • Version curated tables; retain dq_run_summary history.
  • Integrate curated views into hackathon app SQL (scenario_district_impact.sql, etc.).

Operator type whitelist

CASE
  WHEN LOWER(TRIM(operatorTypeId)) IN ('public', 'government') THEN 'public'
  WHEN LOWER(TRIM(operatorTypeId)) = 'private' THEN 'private'
  WHEN operatorTypeId IS NULL OR TRIM(operatorTypeId) IN ('', 'null') THEN 'unknown'
  ELSE 'unknown'  -- quarantine: URLs, JSON, floats logged to dq_facility_profile
END

NFHS percentage parse function

CREATE OR REPLACE FUNCTION parse_nfhs_pct(raw STRING)
RETURNS STRUCT<value: DOUBLE, is_suppressed: BOOLEAN, raw_value: STRING>
RETURN CASE
  WHEN raw IS NULL OR TRIM(raw) = '' THEN named_struct('value', CAST(NULL AS DOUBLE), 'is_suppressed', false, 'raw_value', raw)
  WHEN TRIM(raw) = '*' THEN named_struct('value', CAST(NULL AS DOUBLE), 'is_suppressed', true, 'raw_value', raw)
  ELSE named_struct(
    'value',
    TRY_CAST(REGEXP_REPLACE(TRIM(raw), '[()]', '') AS DOUBLE),
    'is_suppressed', false,
    'raw_value', raw
  )
END;

Geographic Join Layer

The curated join path replaces ad-hoc TRIM/LOWER in application SQL.

dim_facility │ pincode ──► bridge_pincode_district ──► dim_geography.district_id │ └ (fallback) lat/lon ──► nearest dim_geography centroid [flag: geo_fallback] fact_nfhs_district_indicators └──► dim_geography.district_id (via normalized district_name + state_ut) agg_district_supply = GROUP BY district_id FROM dim_facility agg_district_supply_demand_gap = agg_district_supply FULL OUTER JOIN fact_nfhs ...

Sample DQ summary query

INSERT INTO virtue_foundation_curated.dq_run_summary
SELECT
  current_timestamp() AS run_at,
  COUNT(*) AS facility_count,
  AVG(CASE WHEN join_confidence = 'high' THEN 1.0 ELSE 0.0 END) AS pct_high_confidence_join,
  SUM(CASE WHEN operator_type = 'unknown' THEN 1 ELSE 0 END) AS unknown_operator_count,
  SUM(CASE WHEN bed_count IS NOT NULL THEN 1 ELSE 0 END) AS bed_count_populated,
  SUM(CASE WHEN coord_valid THEN 1 ELSE 0 END) AS valid_coord_count
FROM virtue_foundation_curated.dim_facility;
